Front End directions

Frameworks

What are Frameworks

  • Already prepared sets of elements and styles
  • Can provide many features such as:
    • Eeasy and quick responsiveness
    • Layout help
    • Plug-and-use components
    • Customizable features such as color or shape
  • Documentation is always provided for use

Benefits of Frameworks

  • Faster and easier development
  • Responsiveness and compatibility implementation
  • Standardized and uniform components
  • Ready-to-go components that are used often like Navigation bars, headers, footers, etc.

Bootstrap

  • Has a grid layout system

  • Will be used in later back end subjects

Bootstrap Setup

  • Has dependent libraries that it needs to work
  • Component can be grabbed from the components page

Pre-processors

What are Pre-processors

  • They extend the default CSS features and capabilities
  • With them, we can use logic and write statements in our CSS
  • They allow building reusable CSS blocks of code

Benefits of Pre-processors

  • It helps with automating tasks in CSS
  • It makes CSS easily configurable and reusable
  • Reduce the number of lines in our CSS code
  • Ensures backward compatibility for different browsers

Sass

  • Syntactically Awesome Style Sheets
  • Allows the use of variables and logic in the CSS code
  • Implements features such as if-else, while, for, etc.
  • Needs to be compiled to work ( can't run as is )

Sass Setup

  • Can be used and easily implemented by an application
  • Can be installed manually through command line
  • How to get started guide

Less

  • Leaner Style Sheets
  • Inspired by Sass
  • Has similar features to Sass, but a bit differently implemented
  • Can use reusable parts, variables, conditions, etc.

Less Setup

  • Can be installed manually through command line
  • How-To guide

Less Resources

  • Documentation
  • Get Started Blog
  • Features and guide

Less VS Sass comparison

  • Both have very similar features, but with different syntax
  • Variables in Sass start with $ and in less with @
  • Sass is more popular than less
  • less has better error reporting and indicating
  • Both have a bit different implementation for reusable code
